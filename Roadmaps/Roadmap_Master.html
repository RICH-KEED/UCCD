Git Rules — How to Work Without Breaking Each Other
  • 🌿
    Never commit to main directly. Each person works on their own branch: feature/akash-ml-pipeline, feature/abhineet-api-ai, feature/hemant-frontend, feature/pritesh-auth-sla-devops. Create a PR to merge into main at end of each week.
  • 📁
    Never edit someone else's files. If you need a function from another person's file, import it — don't copy it. If you need them to add something, tell them in WhatsApp and they add it to their file.
  • 🔗
    The one shared file is docker-compose.yml. Akash owns Kafka + Postgres sections. Pritesh owns Redis + API sections. Always pull before editing this file and coordinate on WhatsApp before pushing changes to it.
  • 🚨
    Week 1 is the most critical. Akash's db/connection.py and Abhineet's mock API endpoints MUST be pushed by Day 5 of Week 1. If either slips, hemant and Pritesh are blocked. This is the single most important deadline.
  • 🔔
    When you push something others depend on, message the team immediately. "Akash: db/connection.py pushed, docker-compose updated — pull and run make up to test". Don't assume they'll see it.
  • ✅
    End of every week: run make test. Before merging your PR to main, all tests must pass. If Pritesh's tests fail because of your code, fix it before merging.
Demo Flow for Judges — 2 Minutes

This is the exact sequence to demo when presenting. Practice it until it takes exactly 2 minutes.

  • 1
    Login screen → log in as Supervisor. (5 sec)
  • 2
    Supervisor HQ → show live KPI bar (1,284 open, 23 breached, 61 predicted). Point to the "AI PREDICTED" count and say "these haven't breached yet — UCCD flagged them before the deadline." (20 sec)
  • 3
    Escalation queue → show one BREACH row + one PREDICT row. Click the PREDICT complaint. (10 sec)
  • 4
    Complaint Detail → point to emotion arc chart trending downward (red). Point to the teal "🌐 Urdu" language badge next to customer name and say "this complaint came in Urdu, UCCD detected and translated it automatically." Point to AI draft already written. Point to triage panel showing breach probability 82% in red. Show the translation preview below the draft and say "when the agent clicks Send, the customer receives this in Urdu — the agent never had to speak the language." (25 sec)
  • 5
    Log out, log in as Agent. Open My Queue. Show SLA countdowns. Open a complaint, edit AI draft slightly, click Send Response. (20 sec)
  • 6
    Insights Board → show trend chart. Say "47 complaints share the same DNA cluster — same root cause." (15 sec)
  • 7
    Simulation Sandbox → set condition "complaint_type = billing AND amount < 1000", click Run. Show "2,847 complaints would resolve, ₹5.1 Lakh cost saving." (15 sec)
  • 8
    Regulatory Dashboard → show deadline countdowns, point to auto-populated RBI report. (10 sec)
