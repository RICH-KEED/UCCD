Your Responsibility

You own the entire frontend. React + TypeScript app that calls Abhineet's API. You build 8 screens for the demo — the most important ones. All screens use a dark navy theme with teal accents (the design system is defined below). Week 1 you build structure and start with mock data since Abhineet's real API won't be fully ready yet.

Design system colors you must use consistently: Background: #0A1628, Cards: #1A3356, Teal accent: #0D9488, White text: #F8FAFC, Muted text: #64748B, Red alert: #EF4444, Amber warning: #F59E0B.

Week 2 — My Queue + Complaint Detail
WEEK 02
Agent's Two Most Used Screens
Days 8–14
SCREEN 2My Queue

List of assigned complaints sorted by SLA urgency. Each row shows severity color, complaint type badge, customer name, channel icon, SLA countdown timer in red/amber/green based on time remaining.

ComplaintRow SLATimer SeverityBadge FilterPills ChannelIcon
1
Fetch from GET /api/v1/complaints?status=open&limit=50. Show loading skeleton while fetching.
2
Build SLATimer component — takes sla_deadline ISO string, shows live countdown updating every second using setInterval. Color: red if under 4 hours, amber if under 12 hours, green otherwise.
3
Filter pills: All, At Risk (under 12h), Breached (past deadline), New Today. Clicking a pill filters the list client-side.
4
Each row has a View button → navigate to /complaints/{id}. Left border color matches severity (red=high, amber=medium, gray=normal).
API: GET /api/v1/complaints?status=open&assigned_to={agent_id}
SCREEN 3Complaint Detail — The Main Working Screen

Most important screen in the entire product. 3-column layout. Left: customer info + emotion arc chart + linked complaints. Middle: communication history + AI draft. Right: AI triage results + NBA panel.

EmotionArcChart ConversationHistory AIDraftPanel TriagePanel NBAPanel LinkedComplaints SLACountdown
1
Fetch complaint from GET /api/v1/complaints/{id}. Show all fields.
2
Build EmotionArcChart using Recharts LineChart. X-axis = turn number, Y-axis = sentiment score (-1 to +1). Line color: green if positive slope, red if negative. Small sparkline size (200px × 80px).
3
Build ConversationHistory — chat-style bubbles. Customer messages on left (dark card), agent responses on right (slightly lighter). Show timestamp + channel icon per turn.
4
Build AIDraftPanel — show ai_draft_response from API in a textarea that is pre-filled but editable. "Send Response" button calls POST /api/v1/complaints/{id}/respond with edited text.
5
Build TriagePanel (right column) — show complaint_type badge, severity_score as a 0–100 bar gauge (color: green under 40, amber 40–70, red over 70), sla_tier badge, breach_probability as percentage (red if over 70%).
6
Build NBAPanel — hardcode 3 resolution actions for demo: "Issue Refund", "Escalate to Branch", "Close with Explanation" with fake FCR percentages. Click copies action to draft panel.
7
Top header bar: complaint ID + type badge + SLA countdown + status badge + Escalate button (red outline) + Resolve button (teal filled).
8
Add LanguageBadge near customer name using source_language from API (example: any detected language like Hindi, Tamil, Bengali, Marathi, Urdu, English, or non-Indian languages).
9
Sarvam — Show original toggle: add OriginalTextToggle to switch between translated English text and original language text in conversation history.
10
For TranslationPreview, call GET /api/v1/ai/translate-preview?text=...&target=... only when typing pauses for 1 second (debounce 1000ms), not on every keystroke.
API: GET /complaints/{id} · GET /complaints/{id}/history · GET /ai/draft/{id} · POST /complaints/{id}/respond
Week 3 — Supervisor Command Centre
WEEK 03
Live Operations Dashboard
Days 15–21
SCREEN 4Supervisor Command Centre

Full-width no sidebar. Live KPI bar across top. Left 60%: escalation queue table. Right 40%: agent load bars + AI intelligence feed. Volume bar chart at bottom.

KPIBar EscalationQueue AgentLoadBars AIFeed VolumeChart LiveDot
1
KPI bar: 5 stat cards side by side — Open Complaints (white), Breached (red), AI Predicted (amber), FCR Rate (green), Avg Handle Time (teal). Fetch from GET /api/v1/kpis. Poll every 30 seconds.
2
Escalation Queue table — fetch from GET /api/v1/escalations. Columns: Priority badge (BREACH red pill / PREDICT amber pill), Complaint ID, Customer, Type, Time Remaining (red countdown), Agent. Sort by urgency. Refresh every 30 seconds.
3
Agent Load Bars — fetch from GET /api/v1/agents/load. For each agent: name + avatar initials + horizontal progress bar 0–100%. Bar color: green under 60, amber 60–80, red over 80. Show queue count number. Poll every 15 seconds.
4
AI Intelligence Feed — hardcode 3 feed items for demo with different dot colors (red=spike, amber=burnout, teal=opportunity). In Week 5, connect to WebSocket for real-time updates.
5
Volume Chart — Recharts BarChart showing complaint count per hour for last 12 hours. Fetch from GET /api/v1/analytics/trends?window=12h. Bars in teal.
6
Green pulsing "LIVE" dot in header — CSS animation @keyframes pulse, opacity 1→0.3→1, 1.5s infinite.
API: GET /kpis · GET /escalations · GET /agents/load · GET /analytics/trends
