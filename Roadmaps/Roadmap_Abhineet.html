Week 2 — Claude API: NLP Classifier
WEEK 02
Claude-Powered NLP Classification Agent
Days 8–14
AIModule 2 — NLP Classifier Agent

Use Claude API to classify every complaint across 4 dimensions simultaneously: type, product, intent, and regulatory obligation. Structured JSON output only. This runs as a Kafka consumer — reads complaint, classifies, writes back to DB.

Install
pip install anthropic
1
Get API key from console.anthropic.com. Store in .env as ANTHROPIC_API_KEY. Never commit the key.
2
Write agents/nlp_classifier.py — classify_complaint(text: str, bot_slots: dict) -> dict.
3
System prompt: "You are a PSB complaint classifier. Return ONLY valid JSON. No explanation, no markdown."
4
User prompt asks Claude to return: type (fraud/billing/kyc/loans/cards/service/other), type_confidence (0–1), product (savings/current/credit_card/home_loan/personal_loan/fd/insurance), intent (refund/explanation/escalation/closure/legal_threat), regulatory_obligation (banking_ombudsman/rbi_consumer/none).
5
Use model="claude-sonnet-4-6", max_tokens=300. Parse response with json.loads(). If parsing fails, return default classification with type="other" and confidence=0.5.
6
Write NLPConsumer(BaseConsumer) extending Akash's base class. Consume from complaints.inbound, classify, write complaint_type, product_code, intent, regulatory_flag to DB row.
7
Test with 20 different complaint texts. Check Claude always returns valid JSON. Add retry logic with exponential backoff for API rate limits.
Input
  • raw_text — complaint string
  • bot_slots — dict (optional pre-fills)
Output
  • complaint_type
  • type_confidence
  • product_code
  • intent
  • regulatory_obligation
Week 3 — LangGraph Orchestrator
WEEK 03
Multi-Agent Pipeline (All 6 Agents in Parallel)
Days 15–21
AIModule 3 — LangGraph Agent Orchestrator

Wire all 6 agents (NLP Classifier + Akash's 4 ML agents + Root Cause) into a LangGraph graph that runs them in parallel. This is the core pipeline — one complaint in, fully enriched complaint out in under 800ms.

Install
pip install langgraph langchain-anthropic
1
Define ComplaintState TypedDict in agents/state.py — all fields the agents read and write: complaint_id, text, channel, classification, emotion_arc, embedding, cluster_id, severity_score, sla_tier, breach_probability, pre_escalate.
2
Write wrapper functions for each agent that takes state and returns updated state: run_nlp(state), run_emotion(state), run_dna(state), run_severity(state), run_escalation(state).
3
Build the LangGraph: add each agent as a node, add parallel edges from START to all 5 agents simultaneously. Add a final orchestrator node that waits for all 5 and commits the merged state to PostgreSQL.
4
Write agents/root_cause_agent.py — simple version: query DB for complaints in same cluster. If count ≥ 10 and no root_cause_id yet, create a root cause record and link all cluster members. Uses psycopg2 directly (no Neo4j needed for MVP).
5
Compile the graph: pipeline = graph.compile(). Expose run_pipeline(complaint_id: str, text: str, ...) -> ComplaintState function.
6
Integrate with FastAPI: when POST /api/v1/complaints is called, publish to Kafka AND call run_pipeline() async. Return immediately with complaint_id — enrichment happens in background.
7
Time the full pipeline end-to-end. Should be under 2 seconds for MVP (800ms target is for production scale). Log timing for each agent node.
Input
  • Raw complaint from Kafka
  • complaint_id, text, channel, customer_id
Output
  • Fully enriched complaint row in DB
  • All agent outputs written atomically
Week 4 — Draft Response Generator + WebSocket
WEEK 04
AI Draft Response + Real-Time WebSocket Server
Days 22–28
AIModule 4 — Draft Response Generator

Generate a tone-matched draft response for every complaint using Claude. Tone is determined by the emotion arc slope from Akash's agent. Agent edits are tracked for future model improvement.

1
Write services/draft_service.py — generate_draft(complaint_text, emotion_arc, complaint_type, regulatory_flag) -> str.
2
Determine tone: if emotion_arc["arc_slope"] < 0 → "empathetic", elif regulatory_flag → "regulatory_formal", else → "professional".
3
System prompt: f"You are a {tone} PSB complaint officer. Write a response under 120 words. Be specific and action-oriented."
4
Add endpoint: GET /api/v1/ai/draft/{complaint_id} — fetches complaint from DB, generates draft, returns it. Also stores draft in ai_draft_response column.
5
Add endpoint: POST /api/v1/complaints/{id}/respond — agent sends final response. Store in final_response, compute edit_delta (character diff vs draft), set status to "resolved".
FASTAPIModule 5 — WebSocket Server

Real-time push notifications to the supervisor dashboard. When a breach happens or is predicted, the supervisor sees it instantly without refreshing.

1
Write api/websocket.py — ConnectionManager class with connect(ws), disconnect(ws), broadcast(message) methods. Store active connections in a set.
2
Add WS endpoint: ws://localhost:8000/api/v1/ws/supervisor. When a client connects, add to manager. When they disconnect, remove.
3
From the Kafka consumer or SLA check — call manager.broadcast({"type": "breach_predicted", "complaint_id": ..., "probability": ...}). Supervisor dashboard receives this instantly.
4
Message types to support: new_complaint, breach_predicted, breach_occurred, cluster_spike, agent_overload.
5
Test using browser DevTools console: const ws = new WebSocket('ws://localhost:8000/api/v1/ws/supervisor'). Trigger a breach — verify message arrives instantly.
Input (WebSocket)
  • Events from SLA engine or Kafka consumers
Output
  • Real-time JSON push to all connected supervisor clients
